## Installation

```bash
cargo add l402kit
```

Or in `Cargo.toml`:

```toml
[dependencies]
l402kit = "1.1"
```

**Requirements**: Rust 1.75+, Tokio async runtime

---

## Zero-config managed mode

The fastest way to monetize your API. Set your Lightning Address — l402-kit handles invoices, verification, and fee split automatically. You receive **99.7%** of every payment directly to your wallet.

```rust
use axum::{Router, routing::get, Json};
use l402kit::{l402_middleware, Options};
use serde_json::{json, Value};

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/api/data", get(handler))
        .layer(l402_middleware(Options {
            price_sats: 10,
            owner_lightning_address: Some("you@blink.sv".into()),
            ..Default::default()
        }));

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

async fn handler() -> Json<Value> {
    Json(json!({ "data": "premium content" }))
}
```

No Lightning node, no invoice handling, no crypto code.

## Self-hosted mode (bring your own provider)

```rust
use std::sync::Arc;
use l402kit::{l402_middleware, Options, LightningProvider, Invoice, BoxFuture, L402Error};

struct MyProvider;

impl LightningProvider for MyProvider {
    fn create_invoice<'a>(&'a self, amount_sats: u64) -> BoxFuture<'a, Result<Invoice, L402Error>> {
        Box::pin(async move {
            // create invoice with your Lightning node
            Ok(Invoice {
                payment_request: "lnbc...".into(),
                payment_hash: "abc123...".into(),
                macaroon: "eyJ...".into(),
                amount_sats,
            })
        })
    }
}

let app = Router::new()
    .route("/api/data", get(handler))
    .layer(l402_middleware(Options {
        price_sats: 10,
        lightning: Some(Arc::new(MyProvider)),
        ..Default::default()
    }));
```

## `Options`

| Field | Type | Default | Description |
|---|---|---|---|
| `price_sats` | `u64` | **required** | Price per call in satoshis |
| `owner_lightning_address` | `Option<String>` | `None` | Your Lightning Address — enables managed mode |
| `lightning` | `Option<Arc<dyn LightningProvider>>` | `None` | Custom provider. Overrides `owner_lightning_address` |
| `on_payment` | `Option<Box<dyn Fn(L402Token, u64)>>` | `None` | Callback fired after each verified payment |

## `l402_middleware(opts: Options)`

Returns an `axum::middleware::Layer` compatible with axum 0.8+.

### Behavior

| Request | Response |
|---|---|
| No `Authorization` header | `402` + `WWW-Authenticate: L402 macaroon="...", invoice="lnbc..."` |
| Valid `L402 <macaroon>:<preimage>` | Handler executes |
| Invalid or expired token | `401 Unauthorized` |
| Replayed preimage | `401 Token already used` |

### 402 response body

```json
{
  "error": "Payment Required",
  "invoice": "lnbc100n1...",
  "macaroon": "eyJoYXNoIjoiYWJjMTIzIiwiZXhwIjoxNzAwMDAwMDAwfQ==",
  "price_sats": 10
}
```

## `on_payment` callback

```rust
use l402kit::{Options, L402Token};

let app = Router::new()
    .route("/api/data", get(handler))
    .layer(l402_middleware(Options {
        price_sats: 10,
        owner_lightning_address: Some("you@blink.sv".into()),
        on_payment: Some(Box::new(|token: L402Token, amount_sats: u64| {
            println!("payment received: {} sats", amount_sats);
        })),
        ..Default::default()
    }));
```

## Feature flags

```toml
[features]
default = ["axum-middleware"]
axum-middleware = ["dep:axum", "dep:reqwest", "dep:http"]
```

Disable `axum-middleware` to use only the core verification functions without axum or reqwest:

```toml
l402kit = { version = "1.1", default-features = false }
```

---

## Verification

`SHA256(preimage) == paymentHash` is verified locally using the `sha2` crate — no network call on the hot path. Token expiry is checked in the same operation.

---

## Error codes

| Status | Meaning |
|---|---|
| `402` | No payment token — pay the invoice |
| `401` | Invalid token or expired macaroon |
| `401` | Replayed preimage (already used) |
