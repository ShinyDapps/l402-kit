## Installation

```bash
npm install l402-kit
```

**Requirements**: Node.js 18+, Express 4+

## Zero-config managed mode

The fastest way to monetize your API. Set your Lightning Address ‚Äî l402-kit handles invoices, verification, and fee split automatically. You receive **99.7%** of every payment directly to your wallet.

```typescript
import express from 'express';
import { l402 } from 'l402-kit';

const app = express();

app.get('/api/data', l402({
  priceSats: 10,
  ownerLightningAddress: 'you@yourdomain.com',
}), (req, res) => {
  res.json({ data: 'premium content' });
});

app.listen(3000);
```

That's it. No Lightning node, no invoice handling, no crypto code.

## `l402(options)` ‚Äî middleware

Returns an Express `RequestHandler` that enforces L402 payment on the route.

### Options

| Option | Type | Default | Description |
|---|---|---|---|
| `priceSats` | `number` | **required** | Price per call in satoshis |
| `ownerLightningAddress` | `string` | ‚Äî | Your Lightning Address. Enables managed mode ‚Äî no provider needed |
| `lightning` | `LightningProvider` | ‚Äî | Custom Lightning provider. Required if `ownerLightningAddress` is not set |
| `supabaseUrl` | `string` | `SUPABASE_URL` env | Supabase URL for payment logging |
| `supabaseKey` | `string` | `SUPABASE_ANON_KEY` env | Supabase key for payment logging |
| `onPayment` | `(token, amountSats) => void` | ‚Äî | Callback fired after each verified payment |
| `webhookUrl` | `string` | ‚Äî | Your endpoint to receive signed payment events |
| `webhookSecret` | `string` | ‚Äî | HMAC-SHA256 secret for webhook signing |
| `replayAdapter` | `ReplayAdapter` | in-memory | Pluggable replay protection backend |

### Behavior

| Request | Response |
|---|---|
| No `Authorization` header | `402 Payment Required` with BOLT11 invoice + macaroon |
| Valid `L402 <macaroon>:<preimage>` | `next()` ‚Äî handler executes |
| Invalid or expired token | `401 Unauthorized` |
| Replayed preimage | `401 Token already used` |

### 402 response body

```json
{
  "error": "Payment Required",
  "invoice": "lnbc100n1p...",
  "macaroon": "eyJoYXNoIjoiYWJjMTIzIiwiZXhwIjoxNzAwMDAwMDAwfQ==",
  "priceSats": 10
}
```

### WWW-Authenticate header

```
WWW-Authenticate: L402 macaroon="eyJ...", invoice="lnbc..."
```

## Replay protection

### Default ‚Äî in-memory (development)

Built-in. Resets on restart. Suitable for single-process deployments.

```typescript
// No configuration needed ‚Äî enabled by default
app.get('/api', l402({ priceSats: 10, ownerLightningAddress: 'you@yourdomain.com' }), handler);
```

### Redis (production ‚Äî multi-instance)

```typescript
import Redis from 'ioredis';
import { l402, RedisReplayAdapter } from 'l402-kit';

const replay = new RedisReplayAdapter(
  new Redis(process.env.REDIS_URL!),
  86400, // TTL in seconds (24h)
);

app.get('/api', l402({
  priceSats: 10,
  ownerLightningAddress: 'you@yourdomain.com',
  replayAdapter: replay,
}), handler);
```

`RedisReplayAdapter` uses `SET key 1 NX EX ttl` ‚Äî atomic, race-condition free.

## Payment webhooks

Receive a signed event after each payment. Use this to trigger side-effects (provision access, send content, log analytics).

```typescript
import { l402, verifyWebhook } from 'l402-kit';

app.get('/api/data', l402({
  priceSats: 10,
  ownerLightningAddress: 'you@yourdomain.com',
  webhookUrl: 'https://yourapi.com/webhooks/l402',
  webhookSecret: process.env.L402_WEBHOOK_SECRET!,
}), handler);

// Webhook receiver
app.post('/webhooks/l402', express.raw({ type: '*/*' }), (req, res) => {
  const valid = verifyWebhook(
    process.env.L402_WEBHOOK_SECRET!,
    req.body.toString(),
    req.headers['l402-signature'] as string,
  );
  if (!valid) return res.status(401).end();

  const event = JSON.parse(req.body.toString());
  console.log('Payment received:', event.data.paymentHash, event.data.amountSats);
  res.json({ ok: true });
});
```

**Webhook payload:**

```json
{
  "id": "evt_abc123",
  "type": "payment.received",
  "created": 1700000000,
  "data": {
    "endpoint": "/api/data",
    "amountSats": 10,
    "paymentHash": "sha256-of-preimage",
    "ownerAddress": "you@yourdomain.com"
  }
}
```

## `onPayment` callback

Synchronous hook called after each verified payment, before `next()`:

```typescript
app.get('/api/data', l402({
  priceSats: 10,
  ownerLightningAddress: 'you@yourdomain.com',
  onPayment: async ({ macaroon, preimage }, amountSats) => {
    await myAnalytics.track('payment', { amountSats });
  },
}), handler);
```

## Supabase payment logging

Set `SUPABASE_URL` + `SUPABASE_ANON_KEY` in your environment (or pass via options) to log payments automatically.

```typescript
// Via env vars (recommended)
// SUPABASE_URL=https://xxx.supabase.co
// SUPABASE_ANON_KEY=...
app.get('/api', l402({ priceSats: 10, ownerLightningAddress: 'you@yourdomain.com' }), handler);

// Or explicit
app.get('/api', l402({
  priceSats: 10,
  ownerLightningAddress: 'you@yourdomain.com',
  supabaseUrl: 'https://xxx.supabase.co',
  supabaseKey: process.env.SUPABASE_ANON_KEY!,
}), handler);
```

**Payments table schema** (`payments`):

```sql
create table payments (
  id            uuid primary key default gen_random_uuid(),
  payment_hash  text unique not null,  -- SHA256(preimage) ‚Äî safe to store
  endpoint      text,
  amount_sats   integer,
  owner_address text,
  paid_at       timestamptz default now()
);
```

<Note>
`payment_hash` stores `SHA256(preimage)`, not the raw preimage. The preimage is the 32-byte Lightning payment secret ‚Äî its hash is already public in the BOLT11 invoice. This protects your users in case of a database breach.
</Note>

## Standalone utilities

```typescript
import { verifyToken, parseToken, checkAndMarkPreimage } from 'l402-kit';

// Verify a token (returns true/false)
const isValid = await verifyToken('eyJoYXNoIjoi...:deadbeef...');

// Parse token parts
const { macaroon, preimage } = parseToken(token);

// Custom replay check (returns true = first use, false = replay)
const isFirstUse = await checkAndMarkPreimage(preimage);
```

## Types

```typescript
import type { L402Options, LightningProvider, Invoice, L402Token } from 'l402-kit';

interface L402Options {
  priceSats: number;
  ownerLightningAddress?: string;
  lightning?: LightningProvider;
  supabaseUrl?: string;
  supabaseKey?: string;
  onPayment?: (token: L402Token, amountSats: number) => void | Promise<void>;
  webhookUrl?: string;
  webhookSecret?: string;
  replayAdapter?: ReplayAdapter;
}

interface Invoice {
  paymentRequest: string;
  paymentHash: string;
  macaroon: string;
  amountSats: number;
  expiresAt: number;        // Unix ms
}

interface LightningProvider {
  createInvoice(amountSats: number): Promise<Invoice>;
  checkPayment(paymentHash: string): Promise<boolean>;
}
```

## Migration guide ‚Äî v1.1 ‚Üí v1.2

If you use the Supabase integration (`SUPABASE_URL` + `SUPABASE_ANON_KEY`), rename the column in your `payments` table:

```sql
ALTER TABLE payments RENAME COLUMN preimage TO payment_hash;
```

No other changes required. The public API is identical.
